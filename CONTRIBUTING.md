# Contributing

Thank you for your interest in contributing.

This project is intended to be developed in public with a narrow, practical scope. Please favor small, reviewable changes over broad refactors.

## Contribution principles

- prefer explicit behavior over implicit magic
- document assumptions and limitations
- keep rules understandable and testable
- add fixtures for edge cases when adding or changing rule logic
- do not introduce hosted-service assumptions into the core library

## Development flow

1. Open an issue or comment on an existing one.
2. Propose a small scoped change.
3. Add or update tests and fixtures.
4. Update docs when behavior or scope changes.
5. Keep commit history understandable.

## Pull request checklist

- tests added or updated
- docs updated where relevant
- limitations stated clearly
- no secrets or local data committed
- no unnecessary dependency bloat
