# Threat Model (Initial)

This document captures the problem space the project is trying to reduce, not a claim of complete protection.

## Risks the project may help reduce

- avoidable fee anomalies
- unnoticed dust or uneconomic outputs
- incomplete output review before signing
- weak consistency across internal transaction review tools

## Risks not solved by this project

- compromised signing devices
- malicious wallet software outside the analysis boundary
- all forms of address substitution or supply-chain compromise
- final human judgment errors
- Bitcoin network policy changes or miner behavior

## Important note

Analysis output should be treated as advisory safety tooling, not as proof that a transaction is safe.
