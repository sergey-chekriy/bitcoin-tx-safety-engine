# ADR 0001: Favor narrow scope over broad feature surface

## Status
Accepted

## Context
The project is a new public-good proposal and should optimize for a clear, auditable MVP rather than premature breadth.

## Decision
The initial version will prioritize a small ruleset, a small CLI surface, documentation, fixtures, and explicit limitations.

## Consequences
This should improve the chances of shipping a usable first release, but it also means deferring broader integrations and advanced analysis features.
